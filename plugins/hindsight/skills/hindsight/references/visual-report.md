# Phase 3f: Visual Report (only if --viz flag is set)

After Phase 3e wrap-up completes (including writing `last-retro.json`), generate a visual HTML report. The `--viz` flag was set at invocation, so no additional confirmation is needed.

## Steps

1. **Use these inline styles** as the `<style>` block (a warm dark palette with severity-coded accents):
   ```css
   :root { --foreground:#faf8f5; --background:#1c1917; --card:#292524;
     --border:#3d3835; --muted-foreground:#a8a29e; --primary:#84cc16;
     --accent:#ea580c; --sev-critical:#ef4444; --sev-high:#eab308;
     --sev-medium:#38bdf8; --sev-low:#84cc16; }
   body { font-family:sans-serif; color:var(--foreground); background:var(--background);
     max-width:900px; margin:0 auto; padding:2rem; }
   .card { background:var(--card); border:1px solid var(--border);
     border-radius:12px; padding:1.5rem; margin-bottom:1rem; }
   .severity-badge { padding:0.2rem 0.6rem; border-radius:8px; font-size:0.75rem; }
   .severity-badge.critical { color:var(--sev-critical); }
   .severity-badge.high { color:var(--sev-high); }
   .severity-badge.medium { color:var(--sev-medium); }
   .severity-badge.low { color:var(--sev-low); }
   ```

2. **Read the data:** `~/.claude/hindsight/last-retro.json` (just written in Phase 3e). This contains `all_findings`, `friction_scores`, severity counts, trend delta, mode, and window.

3. **Generate a self-contained HTML file** with these requirements:
   - `<!DOCTYPE html>` with `lang="en"` and `<meta charset="UTF-8">`
   - `<meta http-equiv="Content-Security-Policy" content="script-src 'none'">`
   - `<meta name="viewport" content="width=device-width, initial-scale=1.0">`
   - Google Fonts `<link>` for Nunito (400;600;700;800), Nunito Sans (400;500;600), JetBrains Mono (400;600;700). Accept graceful font degradation if unavailable.
   - The CSS from step 1 as an inline `<style>` block (copy verbatim, do not modify)
   - No `<script>` tags anywhere in the output
   - Container: 900px centered (class `container`)
   - **All finding text must be HTML-escaped:** replace `<` with `&lt;`, `>` with `&gt;`, `"` with `&quot;`, `&` with `&amp;`

   **Sections in order:**

   a. **Header:** `<h1>Hindsight</h1>` with `<p class="subtitle">` showing date range, mode, and window.

   b. **Trend delta** (if previous run exists): `<div class="trend-delta">` with improvement/worsening indicators using `<span class="improved">` or `<span class="worsened">`.

   c. **Stats row:** `<div class="stats-grid">` with `<div class="card stat-card">` for each stat: sessions analyzed, findings count, severity breakdown (CRIT/HIGH/MED/LOW counts). Use `.stat-value.critical` for critical count, `.stat-value.good` for improvements.

   d. **Friction sparkline:** `<div class="card sparkline-row">` containing:
      - `<span class="sparkline-label">Friction</span>`
      - `<div class="sparkline-bars">` with one `<div class="bar">` per session. Height = `(score / max_score) * 100` percent. Add class `high` to bars where score >= 5.
      - `<span class="sparkline-avg">avg X.X/10</span>`
      - If `friction_scores` is empty, render `<div class="empty-state">No session data</div>` instead.

   e. **Findings list:** `<section>` with `<h2>Findings</h2>` then `<div class="findings-list">`. For each finding (sorted by severity, CRITICAL first, max 10 in HTML):
      - `<div class="card finding-card">`
      - `<span class="severity-badge [critical|high|medium|low]">[CRIT|HIGH|MED|LOW]</span>`
      - `<div>` with `<div class="finding-title">` and `<p class="finding-desc">`
      - `<span class="disposition [applied|skipped|discussed]"><span class="disposition-dot"></span> [Applied|Skipped|Discussed]</span>`
      - If 0 findings, render `<div class="card empty-state">No cross-session patterns detected in this window.</div>`
      - If findings were truncated (more than 10), add `<p class="truncation-notice">Showing top 10 of N findings. Full list in terminal output.</p>`

   f. **Detail section for top 3 findings** (by severity): `<div class="card finding-detail">` for each, containing:
      - Severity badge + `<h3>` with finding title
      - `<div class="meta-row">` with evidence count, session count, evidence quality
      - `<ul class="evidence-list">` with max 3 `<li>` items per finding: `<span class="evidence-date">` (YYYY-MM-DD only, no paths) + `<span class="evidence-desc">`
      - `<div class="impact-box"><strong>Impact:</strong> [impact text]</div>`

   g. **Footer:** `<div class="report-footer">Generated by hindsight on [YYYY-MM-DD]. Behavioral patterns only, no verbatim content.</div>`

   **Size target:** under 30KB total.

4. **Write** to `/tmp/hindsight-report-YYYYMMDD-HHMMSS.html` (use current timestamp, distinct from `hindsight_*` cleanup glob pattern).

5. **Post-write validation:** Run `tail -c 20 /tmp/hindsight-report-*.html` and verify the output contains `</html>`. If truncated, print a warning and regenerate before saving.

6. **Save locally:**
   - Copy the report to `~/.claude/hindsight/reports/YYYY-MM-DD-hindsight.html` (use the exact filename, not a glob).
   - Delete the `/tmp` working file once the copy is confirmed.

7. **Print result:** Show the local path to the user:
   ```
     Visual report: <path>
   ```
